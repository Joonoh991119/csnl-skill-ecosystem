## SKILL.md Pattern Implementation

### 1. EMBEDDING_DIM Configuration Variable
- **Value**: 1024 (BGE-M3 model dimension)
- **Location**: setup_pipeline.py lines 25-27
- **Usage**: Parameterizes pgvector schema, config files, and validation
- **Status**: ✓ Fully implemented and flexible

### 2. pgvector Schema with Parameterized Dimensions
- **Location**: setup_pipeline.py lines 107-179
- **Tables Created**:
  - papers: Metadata storage
  - chunks: Section-aware text segments
  - embeddings: Vector storage with dimension={EMBEDDING_DIM}
  - bm25_stats: Term frequencies for hybrid search
  - search_results: Query logging
- **Status**: ✓ Completely parameterized, supports multiple embedding dimensions

### 3. Section-Aware Chunking
- **Location**: ingest_from_zotero.py lines 47-123
- **Sections Recognized**: abstract, methods, results, discussion, introduction, conclusion
- **Features**:
  - Preserves section boundaries
  - Propagates section metadata to chunks
  - Token counting for optimization
  - Graceful fallback for non-standard papers
- **Status**: ✓ Production-ready with section identification

### 4. Hybrid Search with Reciprocal Rank Fusion
- **Location**: hybrid_search.py lines 64-226
- **Components**:
  - Dense search: Cosine similarity on pgvector embeddings
  - Sparse search: PostgreSQL full-text search with BM25 ranking
  - RRF fusion: Combines via reciprocal rank formula: 1/(k+rank)
- **Configurable**: dense_weight=0.6, sparse_weight=0.4, k=60
- **Status**: ✓ Complete hybrid pipeline with mathematical combination

### 5. sentence-transformers with MPS Device
- **Location**: ingest_from_zotero.py lines 256-271, hybrid_search.py lines 92-106
- **Model**: BAAI/bge-m3 (1024-dimensional embeddings)
- **Device Strategy**:
  1. Try MPS (Metal Performance Shaders) for M4 Pro
  2. Fallback to CPU if unavailable
  3. No exceptions - graceful degradation
- **Status**: ✓ M4 Pro optimized with error handling

## Architecture

```
Zotero PDFs
    ↓
[PDF Extraction] → extract_text_from_pdf()
    ↓
[Section Detection] → identify_sections()
    ↓
[Section-Aware Chunking] → create_section_aware_chunks()
    ↓
[Embedding Generation] → model.encode() on MPS device
    ↓
[PostgreSQL + pgvector] → vector({EMBEDDING_DIM}) storage
    ↓
[Hybrid Search] ← Query input
    ├─ Dense: cosine similarity
    ├─ Sparse: BM25 ranking
    └─ RRF: reciprocal rank fusion
    ↓
[Ranked Results] → with metadata and scores
```
